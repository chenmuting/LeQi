# 更新日志

## v1.0.20 build 51

- 修复歌单分类页 `_categories_loaded_once` 属性缺失导致的未捕获异常。
- 歌单分类页打开后自动加载 `/playlist/catlist`。
- 歌单分类加载完成后自动加载当前分类歌单，默认加载“全部”分类。
- 从发现页跳转歌单分类时，若分类尚未加载，会先加载分类，再按目标分类加载歌单。
- 优化歌单分类页歌单表固定列宽和最小宽高，降低歌单名、创建者、ID 被挤压的问题。
- 优化通用歌曲模型表格固定列宽、最小高度和横向滚动，确保歌曲、歌手、专辑、时长、ID 可显示。
- API 能力检测中 `/api/v2/artist/songs` 改为可选兼容接口；当前 API 不支持时显示“可选缺失”，不再计入必需接口失败。

## v1.0.19 build 50

- 新增 `scripts/check_public_release_assets.py`，用于校验公开仓库 Release 资产完整性。
- 新增 `scripts/check_build_integrity.py`，用于校验绿色版压缩包和安装包关键文件。
- Release 工作流新增打包完整性检查和公开 Release 资产校验。
- 启动自检中心增强：增加关键模块导入检查、播放后端检查、打包资源检查和 Release `version.json` 检查。
- 歌手页、专辑页、歌单分类页歌曲表迁移到 `QTableView + QAbstractTableModel`。
- 专辑页新增歌曲内搜索。
- 专辑页新增“歌曲加入队列”和“专辑加入队列”。
- 播放队列排序稳定性修复：拖拽、上移、下移、置顶、置底后保持当前播放歌曲索引正确。
- 播放队列移除歌曲时保持当前播放索引正确。

## v1.0.18 build 49

- 新增启动自检中心：检查配置文件、版本文件、目录权限、数据库、本地歌单 JSON、API 地址等关键启动依赖。
- 启动自检报告支持导出到 `logs/startup_check_<timestamp>.json`。
- 新增专辑页：支持搜索专辑、查看专辑详情、加载专辑歌曲、播放、收藏、加入本地歌单。
- 搜索结果右键新增“打开专辑”。
- 歌手歌曲右键新增“打开专辑”。
- 播放队列支持拖拽排序，并新增上移、下移、置顶、置底操作。
- 搜索页歌曲表试点升级为 `QTableView + QAbstractTableModel`，为后续大列表性能优化打基础。

## v1.0.17 build 48

- 新增发现页，聚合搜索历史、热门歌单分类、歌手入口和 API 能力检测。
- 搜索页新增最近搜索历史，可一键重新搜索历史关键词。
- 搜索结果支持将已加载歌曲批量加入指定本地歌单。
- 歌手页新增歌手详情、歌手简介展示。
- 歌手页新增已加载歌曲内搜索。
- 歌单分类页按 `/playlist/catlist` 的 category 分组显示分类标签。
- 新增 API 能力检测：搜索、播放链接、歌单分类、分类歌单、歌手搜索、歌手歌曲。
- 保留 v1.0.16 的启动闪退修复，新增发现相关功能不在启动阶段主动请求网络接口。

## v1.0.16 build 47

- 修复应用打开时高概率闪退的问题。
- 歌单管理页不再在构造阶段立即读取本地歌单、用户歌单缓存和数据库，改为页面显示后异步刷新。
- 歌单分类页不再在构造阶段立即请求 `/playlist/catlist`，改为页面显示后加载。
- 主题加载失败时记录日志并跳过，不再导致应用退出。
- 启动后台任务延迟到主窗口创建完成后执行，降低启动阶段资源竞争。
- 启动失败时显示错误弹窗并写入 `logs/error.log`，避免用户只看到窗口一闪而过。
- 本地歌单 JSON 损坏时自动重命名为 `.broken_<timestamp>` 并重建空数据。

## v1.0.15 build 46

- 新增歌单分类页，调用 `/playlist/catlist` 获取分类，使用 `/top/playlist` 按分类分页浏览网友精选歌单。
- 搜索页支持分页加载更多，搜索结果追加并按 song_id 去重。
- 新增歌手页：支持搜索歌手、分页加载歌手歌曲、播放、收藏、加入本地歌单。
- 搜索结果右键新增“查看歌手全部歌曲”。
- 本地歌单新增 `category`、`tags`、`description` 字段，旧数据自动兼容为“默认”分类。
- 歌单管理页支持按本地歌单分类筛选，并可编辑分类、标签和说明。
- 侧边栏改为分组显示：播放、资料库、工具、设置。

- 本地音乐启动扫描延迟到页面显示后执行。
- 表格批量渲染时暂停 UI 更新和信号，降低大列表刷新卡顿。
